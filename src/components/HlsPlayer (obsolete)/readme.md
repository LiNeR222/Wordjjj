# Показ модального окна при ошибке авторизации 

# Бесплатное видео

**time_not_reg: 10, time_not_pay: 20 (не используется в бесплатных видео)**

### Неавторизованный пользователь может:
- получить `playlist.m3u8`
- получить `playlist0.ts` (10 секунд),
- `playlist1.ts` возвращает 401

### Авторизованный пользователь может:
- получить `playlist.m3u8`
- получить `playlist0.ts` и все следующие

---

# Платное видео

**time_not_reg: 10, time_not_pay: 20**

### Неавторизованный пользователь может:
- получить `playlist.m3u8`
- получить `playlist0.ts` (10 секунд),
- `playlist1.ts` и следующие возвращает 401

### Авторизованный пользователь может:
- получить `playlist.m3u8`
- получить `playlist0.ts` и `playlist1.ts` (20 секунд)
- `playlist2.ts` и следующие (>20 секунд) возвращает 402


# Схема обработки ошибки авторизации

- При возникновении ошибки загрузки фрагмента, hls loader `custom-loader.ts` сохраняет ошибку в хранилище плеера `playerStore.failedFragments`.

- если фрагмент нулевой, то vime player не вызывает ошибку vmError при ошибке загрузки фрагмента и поэтому приходится делать подписку на событие `onVmPlaybackReady` и обрабатывать ошибку вручную.

- В момент, когда время проигрывания видео дойдет до фрагмента, который вызвал ошибку, hls loader вызывает ошибку vmError и функцию `handleHlsError` из `handle-hls-errors.ts`.

- Функция `handleHlsError` получает ошибку из хранилища плеера и вызывает функцию `handleAuthError` из `handle-auth-error.ts`.

- Функция `handleAuthError` в зависимости от типа ошибки выполняет действия для решения проблемы авторизации. Если запрос успешен, то hls loader перезагружает фрагмент с помощью функции `load` из `custom-loader.ts`, сохраненной в хранилище плеера `playerStore.failedFragments`.

- Если запрос не успешен, то снова выводится модальное окно с ошибкой авторизации. Если нулевой фрагмент по таймеру, если нет, то при возобновлении проигрывания видео.
